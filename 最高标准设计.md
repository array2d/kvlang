# kvlang 最高标准设计

---

## 一、地址空间

kvlang 以 kvspace 树形路径为统一地址空间。路径分四域：

```
/src/{pkg}/{name}         源码文本
/func/{pkg}/{name}        编译后函数（签名 + 指令树）
/op/{backend}/func/{name} 基础算子注册
/vthread/{vid}/           元线程栈帧（运行时）
```

节点间支持软链（value 以 `->` 开头），使路由透明重定向，kvspace 是带链的有向树。

kvspace 存储两类数据：**基础数据类型**（int、float、bool、string 等）和 **tensor 元数据**（shape、dtype、指向扩展存储的句柄）。tensor 的完整数据不在 kvspace 中，而在扩展存储位置：

| 扩展位置 | 典型数据 |
|---------|---------|
| 集群节点共享内存 | 大张量、激活值（heap-plat 管理生命周期） |
| GPU 显存 | 计算张量（op-plat 在设备侧持有句柄） |
| 文件系统 / 对象存储 | 模型权重、检查点、数据集 |

程序员通过统一 kvspace 路径读写任何 tensor，不感知数据的物理位置。

## 二、统一指令形式

kvlang 只有一层 IR，同一语法同时承担四种职能：

| 职能 | 形式 |
|------|------|
| VM 可执行指令 | `add(A, B) -> C` |
| 高级函数定义 | `def gemm(...) -> (...) { ... }` |
| 编译器数据流分析 | `->` / `<-` 即 use-def 链，无需构造 SSA |
| 人类可读源码 | 纯文本，无编号寄存器，无 phi 节点 |

每条语句的统一形式：

```
opcode(reads...) -> writes...
```

控制流不例外：`br(./cond, t, f)`，`call(pkg/func, args...)`，`return(./v)`。

但控制流，我们提供了高级语法糖，if else、label

## 三、执行模型

**PC 是 KV 路径字符串，不是整数。**

```
label `then` → /vthread/tid/[0,0]/then/[0,0]
goto(then)   → PC = funcRoot + "/then/[0,0]"   # 纯字符串拼接，零查表
```

label 块即无参函数，控制流统一为 `call` / `return` 两条原语。
调用 = 在 kvspace 创建子树；返回 = 清理子树、恢复父 PC。

程序执行的每个栈帧始终可见于 kvspace，完全透明可监控。

**底座是分布式的，语言是单线程的。** heap-plat 管理 shm 生命周期，op-plat 消费 GPU 指令，VM 多 worker 并行调度——程序员只写数据流箭头，不感知设备拓扑。
