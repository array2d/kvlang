# example 最高标准设计

---

## 零、三条设计定律

**定律 1 — 规格即测试**  
语言规格中每一条语义规则，必须有至少一个 .kv 文件或断言组专门验证该规则；且期望输出精确到字节。语义规则未被测试 = 该规则未完成。

**定律 2 — 对立等价类必须双覆盖**  
凡有"真/假"结果的操作（比较、逻辑、条件分支），必须同时测试产生 true 和产生 false 的输入等价类。只测 true 路径是单侧覆盖，等同未测试。

**定律 3 — 验证是字节级的，不是退出码级的**  
所有有 print 输出的文件必须用 `check_out` 逐行匹配；`check_exit 0` 只允许用于纯副作用（tensor 创建/删除等无 stdout 的操作）。

这三条定律是本套件与"跑通即算测试"的根本区别。任何违反均视为设计缺陷，而非实现缺陷。

## 一、测试金字塔（按目的分层，非按大小分层）

```
                ┌──────────────┐
                │  P3: 展示层   │  算法展示——证明语言可表达真实程序
                ├──────────────┤
                │  P2: 语义层   │  语义不变量——每条语义规则有专属测试
                ├──────────────┤
                │  P1: 原语层   │  内置操作——每操作 × 所有等价类
                ├──────────────┤
                │  P0: 冒烟层   │  管道冒烟——整条 pipeline 可跑通
                └──────────────┘
```

| 层级 | 核心问题 | 失败意味着 |
|------|---------|-----------|
| P0 | "编译器+运行时能跑一个 print 语句吗？" | 整个系统无法使用 |
| P1 | "每个内置操作在所有输入等价类下输出正确吗？" | 某操作的某类输入结果错误 |
| P2 | "kvlang 的每条语义规则被实现正确了吗？" | 语言行为不符合语言规格 |
| P3 | "kvlang 能自然表达真实算法吗？" | 语言表达力不足 |

旧设计的 T1/T2/T3（≤25行/≤80行/≤200行）是文件大小分类，不是测试目的分类，被废除。

## 三、等价类覆盖模型（P1 专用）

P1 的完备性标准不是"有一个测试"，而是**所有输入等价类都被覆盖**。

### 等价类划分规则

对同一操作，若两个输入在语义上属于同一类（结果相同），则一个文件覆盖整类。不同等价类**必须**各有独立测试或在同一文件内以独立 case 区分。

每个等价类测试需用 **H（Happy）** 和 **E（Edge）** 两个维度标注：
- **H**：该等价类的典型输入（远离边界）
- **E**：该等价类的边界输入（如相等值、零值、最大值）

### P1 比较操作等价类矩阵

| 操作 | 等价类 | H 示例 | E 示例（边界）| 验证输出 |
|------|--------|--------|-------------|---------|
| `==` | 相等→true | 5==5 | 0==0 | `true` |
| `==` | 不等→false | 5==6 | 0==1 | `false` |
| `!=` | 不等→true | 5!=6 | 0!=1 | `true` |
| `!=` | 相等→false | 5!=5 | 0!=0 | `false` |
| `<`  | 严格小于→true | 3<7 | 6<7 | `true` |
| `<`  | 相等→false | 5<5 | 0<0 | `false` |
| `<`  | 大于→false | 7<3 | 7<6 | `false` |
| `>`  | 严格大于→true | 7>3 | 7>6 | `true` |
| `>`  | 相等→false | 5>5 | 0>0 | `false` |
| `>`  | 小于→false | 3>7 | 6>7 | `false` |
| `<=` | 小于→true | 3<=7 | 6<=7 | `true` |
| `<=` | 相等→true | 5<=5 | 0<=0 | `true` |
| `<=` | 大于→false | 7<=3 | 7<=6 | `false` |
| `>=` | 大于→true | 7>=3 | 7>=6 | `true` |
| `>=` | 相等→true | 5>=5 | 0>=0 | `true` |
| `>=` | 小于→false | 3>=7 | 6>=7 | `false` |

**关键要求**：`==` 和 `!=` 各需 2 个等价类；`<`/`>`/`<=`/`>=` 各需 3 个等价类（边界相等是独立类）。

### P1 逻辑操作等价类矩阵

| 操作 | 等价类 | 验证输出 |
|------|--------|---------|
| `&&` | T&&T | `true` |
| `&&` | T&&F | `false` |
| `&&` | F&&T | `false` |
| `&&` | F&&F | `false` |
| `\|\|` | T\|\|T | `true` |
| `\|\|` | T\|\|F | `true` |
| `\|\|` | F\|\|T | `true` |
| `\|\|` | F\|\|F | `false` |
| `!` | !T | `false` |
| `!` | !F | `true` |

**关键要求**：`&&` 需覆盖全部 4 个 truth-table 行，缺一即不完备。

### P1 算术边界矩阵

| 操作 | H 场景 | E 场景（必须有）|
|------|--------|--------------|
| `+` | 正+正 | 0+x=x |
| `-` | 正-小 | 正-大=负数 |
| `*` | 正×正 | x×0=0 |
| `/` | 整除 | 浮点除（不整除）|
| `pow` | 2^3=8 | x^0=1 |
| `sqrt` | √4=2 | √1=1 |
| `abs` | abs(5)=5 | abs(-5)=5（负数） |
| `neg` | neg(5)=-5 | neg(-5)=5（双重取反）|
| `max` | max(3,7)=7 | max(5,5)=5（相等）|
| `min` | min(3,7)=3 | min(5,5)=5（相等）|

### P1 类型转换等价类矩阵

| 转换 | 等价类 | 输入 | 期望 |
|------|--------|------|------|
| float→int | 截断（向零）正数 | 3.7 | 3 |
| float→int | 截断（向零）负数 | -3.7 | -3 |
| int→float | 整数变浮点 | 3 | 3.0 |
| to_bool | 零→false | 0 | false |
| to_bool | 非零→true | -1 | true |

**关键要求**：float→int 必须有负数截断方向验证（向零还是向负无穷是语义问题）。

## 四、语义规则矩阵（P2 专用）

P2 测试**系统行为是否符合语义规约**，而非单个操作的值。每条规则编号全局唯一，对应至少一个断言。

### 4.1 控制流语义规则（CF 系列）

| 规则 | 语义规约 | 测试方法 | 必须验证的反面 |
|------|---------|---------|--------------|
| CF-1 | if(true) → then 分支执行 | 输出 "taken" | else 输出不出现 |
| CF-2 | if(false) → else 分支执行 | 输出 "else" | then 输出不出现 |
| CF-3 | if(false) 无 else → 继续执行后续指令 | 后续 print 正常输出 | 程序不中止 |
| CF-4 | while(false 初始) → 循环体零次 | 累加器=初始值 | 不是"执行一次再判断" |
| CF-5 | while 循环体执行恰好 N 次 | 精确计数 | 不是 N-1 或 N+1 |
| CF-6 | 嵌套 if：外层 false 时内层不执行 | 外 false+内 true → 内输出不出现 | 防止外层被短路跳过 |
| CF-7 | 嵌套 if：外层 true → 内层独立判断 | 外 true+内 false → 走内 else | |
| CF-8 | while 内嵌 if：每轮独立判断条件 | 计数器在条件翻转前后行为正确 | |
| CF-9 | 递归基础情形：直接返回，不再递归 | fact(0)=1、fib(1)=1 | |
| CF-10 | 递归步骤：正确调用自身并组合结果 | fact(5)=120、fib(6)=8 | |
| CF-11 | goto 跳转：只执行目标块，跳过中间指令 | 中间输出不出现 | |
| CF-12 | br(true, L1, L2) → L1；br(false, L1, L2) → L2 | 两种条件分别验证 | L2 在 true 时不执行 |

CF-1 和 CF-2 是**对立等价类**，缺一不可；CF-4 的"零次"和"N次"是对立等价类，缺一不可。

### 4.2 调用/返回协议规则（CR 系列）

| 规则 | 语义规约 | 测试文件 | 必须验证 |
|------|---------|---------|---------|
| CR-1 | 参数正确绑定到形参 | single_call.kv 或 caller.kv | fn(A=5) 内部 ./A==5，输出 5 |
| CR-2 | 返回值正确写入调用方写槽 | single_call.kv | result = fn() 后 ./result == 预期 |
| CR-3 | 多参数各槽独立 | multi_arg.kv | fn(A,B,C)→A+B+C 三者全部正确 |
| CR-4 | 多返回值各槽独立 | multi_ret.kv | (S,D,P)=fn(7,3)→S=10,D=4,P=21 全部输出 |
| CR-5 | 嵌套调用：子帧不污染父帧局部变量 | nested_call.kv | outer 调 inner 前后 outer 局部变量不变 |
| CR-6 | 菱形调用：共享被调函数不冲突 | diamond.kv | A→B→D 和 A→C→D 各自结果正确 |
| CR-7 | 调用链：结果正确传播 N 层 | chain.kv | f1→f2→...→fN 最终值正确 |

CR-4 是当前严重缺口：`multi_ret.kv` 未 print 任何返回值。

### 4.3 帧隔离规则（SI 系列）

| 规则 | 语义规约 | 测试文件 | 验证方式 |
|------|---------|---------|---------|
| SI-1 | 同名参数在不同帧互不干扰 | scope_isolation.kv | fn(x=5) 和 fn(x=10) 各自返回正确值 |
| SI-2 | 递归时每层帧独立存储局部变量 | tco_depth.kv 或 tailrec 文件 | fib(N) 内部各层 n 值不相互覆盖 |
| SI-3 | 多返回值槽绑定不交叉 | scope_isolation.kv | (A,B)=fn() 且 A≠B 时两者均正确 |

### 4.4 TCO 规则（TCO 系列）

| 规则 | 语义规约 | 测试文件 | 验证方式 |
|------|---------|---------|---------|
| TCO-1 | 尾递归不栈溢出 | tco_depth.kv | sum(10000)=50005000 在合理时间内完成 |
| TCO-2 | TCO 后局部变量仍正确 | tailrec_factorial.kv | fact(10)=3628800 |

## 六、文件格式标准

### 6.1 文件头（强制）

```kv
# <name>: <一句话描述，格式"验证 X 在 Y 条件下产生 Z">
# 层级: P<0|1|2|3>
# 规则: <操作标签 | CF-N | CR-N | SI-N | TCO-N>
# 等价类: <H-<类名> | E-<边界名>>（P1 专用）
# 期望输出:
#   <逐行精确，含空格和换行顺序>
```

`规则` 字段与 §四 矩阵编号一一对应，使文件直接可追溯到语义规则。

### 6.2 文件体约定

```kv
str.set("kvlangrun") -> ./term      # 必须第一行
def fn(params) -> (returns) { }    # 函数定义
fn(args) -> ./result               # 顶层调用
print("KEY =", ./result)           # 输出验证
```

### 6.3 命名约定

| 层级 | 命名规则 | 示例 |
|------|---------|------|
| P1 比较 | `<op>_<class>.kv` | `lt_true.kv`、`lt_eq_false.kv`、`lt_gt_false.kv` |
| P1 逻辑 | `<op>_<TF><TF>.kv` | `and_tt.kv`、`and_tf.kv`、`or_ff.kv` |
| P1 算术边界 | `<op>_<variant>.kv` | `mul_zero.kv`、`pow_zero.kv`、`abs_neg.kv` |
| P2 | `<rule_prefix>_<name>.kv` | `si_isolation.kv`、`tco_depth.kv` |
| P3 | `<algorithm>.kv` | `fibonacci.kv` |

## 七、run.sh 验证规范

### 7.1 强制要求

1. 每个有 print 输出的 .kv 文件**必须**有对应 `check_out`；禁止用 `check_exit 0` 代替
2. P2 文件的所有断言输出行均须验证（不允许只验证最后一行 ALL DONE）
3. P3 fizzbuzz 等多行输出文件须多行 `check_out` 验证
4. 每 section 标注其覆盖的规则范围（如 `§12 P1 compare 等价类覆盖`）

### 7.2 新文件接入规范

1. .kv 文件头的 `# 期望输出:` 与实际运行结果对齐
2. run.sh 对应 section 新增 `check_out`
3. 本文档对应矩阵中将规则状态标记完成

## 九、禁止项

| 编号 | 禁止 | 理由 |
|------|------|------|
| E1 | 只测 true 路径，不测 false 路径（单侧覆盖）| 违反定律 2，等同未测试 |
| E2 | 用 `check_exit 0` 验证有 print 输出的文件 | 违反定律 3 |
| E3 | `# 期望输出:` 注释与实际输出不一致 | 注释即规约，不一致则规约失效 |
| E4 | 一个文件测多个无关联特性 | 失败时无法定位是哪条规则出了问题 |
| E5 | 文件互相依赖（A.kv import B.kv）| 一个文件失败会连带另一个 |
| E6 | 输出含非确定性内容（时间、地址、随机数）| 无法 hardcode 期望输出 |
| E7 | 文件写全局路径（`/vthread/...`）| 侵犯引擎命名空间 |
| E8 | 跳过等价类 E（边界值），只写 H（典型值）| 边界值是编译器 bug 最高发地带 |
| E9 | P3 文件只验证最后一行 | fizzbuzz 每一行都是语义断言 |
| E10 | 新语义规则上线但不在 §四 矩阵中添加对应规则和测试 | 新特性无测试 = 未完成 |
