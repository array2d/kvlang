# example 最高标准设计

---

## 零、一句话定位

**example 目录是 kvlang 语言特性的地面真相（ground truth）**：  
凡语言特性在此无对应可运行示例并经 run.sh 验证，即视为未完成。

工业标准参照：Rust by Example / Go Tour / Zig by Example——每个示例独立可运行，期望输出内嵌文件，覆盖系统化。

---

## 一、目录分类法（三层覆盖体系）

```
example/kvlang/
├── builtin/          T1 最小示例   单一语言特性，最短代码，精确期望输出
│   ├── arith/        算术运算
│   ├── compare/      比较运算
│   ├── logic/        逻辑运算
│   ├── cast/         类型转换
│   ├── print/        IO 输出
│   ├── call/         函数调用
│   ├── call_block/   控制流（语法糖形式）
│   └── native/       native 路径（无 print，验 exit 0）
├── controlflow/      T2 综合示例   控制流原语全覆盖 + 嵌套组合，精确期望输出
└── algo/             T3 算法示例   真实算法程序，展示语言设计力，精确期望输出
```

### 各层职责

| 层级 | 单文件行数 | 函数数 | 核心价值 |
|------|-----------|-------|---------|
| T1 | ≤ 25 | 1 | 最小可运行验证；学习入口；回归最快 |
| T2 | ≤ 80 | 2–6 | 特性组合正确性；边界值；嵌套场景 |
| T3 | ≤ 200 | ≤ 20 | 语言表达力；算法可读性；综合集成 |

## 二、示例文件格式标准

每个 `.kv` 文件必须满足：

### 2.1 文件头注释（强制）

```kv
# <name>: <一句话描述>
# 验证: <逗号分隔的语言特性标签>
# 期望输出:
#   <line1>
#   <line2>
```

- `验证:` 取值来自 §三 覆盖矩阵的特性标签
- `期望输出:` 逐行与 stdout 匹配（run.sh 以 `check_out` 验证）
- T1/T2 必须有 `期望输出:`；T3 至少一行关键输出

### 2.2 执行环境设置（强制第一行）

```kv
str.set("kvlangrun") -> ./term
```

必须是第一条顶层指令。所有 print 输出依赖此 terminal 绑定。

### 2.3 示例体约定

```
┌──────────────────────────────────────┐
│  def func(params) -> (returns) { }   │  ← 函数定义（可多个）
│                                      │
│  str.set("kvlangrun") -> ./term      │  ← 终端绑定（必须第一行）
│  func(args) -> ./result              │  ← 顶层调用
│  print("key =", ./result)            │  ← 输出验证
└──────────────────────────────────────┘
```

- 所有变量输出格式：`print("KEY =", ./val)` → stdout: `KEY = value`
- 参数值使用字面量（不依赖外部状态）
- 输出行数固定，无条件分支输出

### 2.4 示例独立性（强制）

- 每个文件完整自包含（不 import 其他 .kv 文件）
- 执行前必须 `kvlang kvspace clear`（run.sh 确保）
- 无运行时副作用残留（函数体不写全局路径）

## 七、禁止项

| 编号 | 禁止 | 理由 |
|------|------|------|
| E1 | 示例文件相互依赖（A.kv load B.kv） | 独立性是测试可靠性前提 |
| E2 | 非确定性输出（随机、时间戳、地址） | 期望输出必须可 hardcode |
| E3 | 文件中写全局绝对路径（`/vthread/...`） | 侵犯引擎命名空间；破坏隔离 |
| E4 | `期望输出:` 与实际 print 输出不匹配 | 注释即规约 |
| E5 | T3 示例长于 200 行 | 过长示例是设计失误，分解函数 |
| E6 | 同一特性在 T1 和 T3 的 T3 版本取代 T1 版本 | T1 必须保留最小示例 |
| E7 | 不加 `str.set("kvlangrun") -> ./term` 却使用 print | print 无终端则静默丢弃，测试失效 |
| E8 | run.sh 中使用 `check_exit 0` 替代 `check_out "pattern"` 来验证有输出的文件 | exit 0 不验证输出正确性 |
