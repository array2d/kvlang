# parser 最高标准设计

> 父节点：根目录 `最高标准设计.md` → `internal/ast/最高标准设计.md` → 本文档

---

## 一、标准前端流水线

无论语言是否行导向，标准编译器前端的数据流只有一条：

```
io.Reader
    │
    ▼  Scan(src string) → []Token         词法分析：字符 → Token
    │
    ▼  parser.ParseCode(r) → (*ast.File, []Diagnostic, error)    语法分析：Token → AST
```

行导向只是语法规则的一种，不是简化流水线的理由。

## 二、目标架构

### 2.1 数据流

```
io.Reader
    │
    ▼ Scan(src string) → []Token
      · 跳过空白（含换行）和 # 注释
      · 产生平坦 Token 流，末尾含 EOF 哨兵
    │
    ▼ parser{tokens []Token, pos int, errors []Diagnostic}
      · peek() / advance() / expect(kind)
      · 递归下降，消费 Token 驱动结构
      · 块深度由消费 LBrace/RBrace Token 自然追踪
      · 不碰原始字符串，不按行切割
    │
    ▼ *ast.File
```

### 2.2 Token 与源码位置

每个 Token 携带源码位置，错误信息才能指向行列：

```go
type Pos struct {
    Line int // 1-based
    Col  int // 1-based
}

type Token struct {
    Kind  Kind
    Value string
    Pos   Pos
}
```

```go
// Scan 将整个源字符串扫描为平坦 Token 流，末尾附 EOF 哨兵。
func Scan(src string) []Token
```

不存在 `Tokenize` 别名，不存在 `ParseLine` 字符串入口。

### 2.3 Parser 结构与错误收集

```go
// Diagnostic 表示一条解析错误。
type Diagnostic struct {
    Pos     Pos
    Message string
}

type parser struct {
    tokens []Token
    pos    int
    errors []Diagnostic // 积累所有错误，不在第一个错误处停止
}

func (p *parser) peek() Token         // 当前 Token，不消费
func (p *parser) advance() Token      // 消费并返回当前 Token
func (p *parser) eat(k Kind) bool     // 若匹配则消费，返回是否成功
func (p *parser) expect(k Kind) Token // 消费；不匹配则追加 Diagnostic，执行 error recovery
```

`expect` 不匹配时追加错误并跳过当前 token（error recovery），解析不中断，一次返回全部错误。

### 2.4 Token 类型

| Kind | 用途 |
|------|------|
| `EOF` | 文件结束哨兵，消除 pos 越界检查 |
| `While` | 关键字统一化，与 If/For/Return 对称 |
| `Colon` | 块标签 `label: {` 的分隔符 |
| `Newline` | 语句分隔符；行导向语言中无 Arrow 终止的指令靠换行终止 |

### 2.5 核心解析函数

```
ParseCode(r) / ParseFile(path) → *ast.File
  └─ parseFile()
       ├─ parseFunc()     → ast.Func            (遇到 "def")
       └─ parseInst() → *ast.Instruction    (顶层调用)

parseFunc()
  ├─ 消费 "def" + Ident（函数名）
  ├─ parseFuncSig() → ast.FuncSig    // 直接解析到结构化类型，不经字符串
  └─ expect(LBrace) → parseBody() → expect(RBrace) → ast.Func

parseBody() → []ast.Stmt             (消费直到 RBrace 或 EOF)
  └─ parseStmt()
       ├─ "if"        → parseIf()
       ├─ "for"       → parseFor()
       ├─ "while"     → parseWhile()
       ├─ "break"     → ast.BreakStmt
       ├─ "continue"  → ast.ContinueStmt
       ├─ Ident + ":" → parseBlockLabel()
       └─ 其他         → parseInst()  // 直接在 token 流上解析，无中间 buffer

parseIf()
  ├─ 消费 "if" → expect(LParen) → parseCondInst() → ast.IfStmt.Cond *Instruction
  ├─ eat(LBrace) → parseBody() → Then → expect(RBrace)
  └─ 若下个是 "else" → 消费 else + LBrace → parseBody() → Else + expect(RBrace)

parseWhile()
  ├─ 消费 "while" → expect(LParen) → parseCondInst() → ast.WhileStmt.Cond *Instruction
  └─ eat(LBrace) → parseBody() → expect(RBrace)

parseBlockLabel()
  └─ 消费 Ident + ":" → eat(LBrace) → parseBody() → expect(RBrace) → ast.BlockStmt
```

块深度由消费 `LBrace`/`RBrace` 自然追踪，不需要 `strings.Count`。

### 2.6 parseInst / parseCondInst / parseFuncSig

```go
// parseInst 在 token 流上直接解析一条指令，到 Newline/RBrace/EOF 为止。
// 纯递归下降，无中间 token buffer，无启发式停止条件。
func (p *parser) parseInst() *ast.Instruction

// parseCondInst 解析 if/while/for 括号内的条件，直接构造 *ast.Instruction。
// 调用时 peek() 为 LParen，内部消费 LParen；返回后已消费 RParen。
func (p *parser) parseCondInst() *ast.Instruction

// parseFuncSig 消费 def ... 到 LBrace 之前，直接构造 ast.FuncSig。
// 不经中间字符串，不需要 tokensToSig。
func (p *parser) parseFuncSig() ast.FuncSig
```

`IfStmt.Cond` / `WhileStmt.Cond` 均为 `*ast.Instruction`，不在 parser 层存储条件字符串。

## 三、文件职责

| 文件 | 职责 | 禁止 |
|------|------|------|
| `scanner.go` | `Scan(src) → []Token`；`Token{Kind,Value,Pos}` / `Kind` / `Diagnostic` 定义 | `import ast`；语法判断 |
| `inst.go` | `(p *parser).parseInst()` / `parseCondInst()` — 直接在 token 流上解析指令 | 独立 slice 传参；操作原始字符串 |
| `stmt.go` | `parseBody` / `parseStmt` / `parseIf` / `parseFor` / `parseWhile` / `parseBlockLabel` | `strings.Count/Index` 做结构判断 |
| `parser.go` | `parser` 结构体；`ParseFile` / `ParseCode` / `parseFile` / `parseFunc` / `parseFuncSig` | — |

### 包内单向依赖

```
parser.go → stmt.go → inst.go → scanner.go
```

parser 包只 import `kvlang/internal/ast`，不 import kvspace、lower 或任何运行时包。

## 四、AST 节点映射

| 语法结构 | AST 节点 |
|----------|----------|
| 单行指令 | `*ast.Instruction`（Reads []Operand, Writes []WriteSlot） |
| 函数定义 | `ast.Func`（Sig FuncSig, Body []Stmt） |
| if/else  | `*ast.IfStmt`（Cond *Instruction） |
| for      | `*ast.ForStmt`（Iter ast.Operand） |
| while    | `*ast.WhileStmt`（Cond *Instruction） |
| break    | `*ast.BreakStmt` |
| continue | `*ast.ContinueStmt` |
| 块标签   | `*ast.BlockStmt`（Body []Stmt） |
| 文件     | `*ast.File`（Funcs []Func, TopLevelCalls []*Instruction） |

## 五、禁止项

- 禁止 parser 包 import 任何内部包（除 `kvlang/internal/ast`）
- 禁止解析阶段访问 kvspace
- 禁止函数体内嵌套函数定义
- 禁止 `strings.Count` / `strings.Index` 做结构判断（靠 Token 类型）
- 禁止在 Token 流之前按行切割为 `[]string`
- 禁止 `Tokenize` / `ParseLine` 等字符串入口（已删除，不得重建）
- 禁止将 token 先收集为 `[]Token` 再转调另一函数解析（`collectInstTokens` 模式）
- 禁止 `tokensToSig() string`：签名必须直接解析为 `ast.FuncSig`，不经字符串重建
- 禁止为 KV 签名单独维护字符串解析路径（`signature.go` 独立解析器）：统一用 `FuncSig`
