# lower 最高标准设计

---

## 零、一句话定位

**lower 是 kvlang 的编译器降级 pass**：将 `*ast.File` 变换为另一个 `*ast.File`，
消除高级控制流节点（`IfStmt`/`WhileStmt`），展开嵌套表达式，
使输出可被 kvcpu **直接解释执行**。

纯函数：无副作用，无 I/O，无 kvspace 访问。输入输出类型相同，可串联、可测试、可证明。

---

## 一、输入 / 输出不变式

### 输入（来自 parser）

- `Func.Body` 可含任意 `Stmt`：`*Instruction`、`*IfStmt`、`*WhileStmt`、`*ForStmt`、`*BlockStmt`
- `*Instruction.Expr` 可为多层嵌套的表达式树（如 `a + b * c`）

### 输出（交给 layoutcode.WriteFunc）

| 不变式 | 说明 |
|--------|------|
| 无 `*IfStmt` / `*WhileStmt` | 全部展开为 `*BlockStmt` + `br` 指令 |
| 无嵌套表达式 | 每条 `*Instruction.Expr` 的所有 `Args` 均为叶节点（kvcpu 可直接执行） |
| `*ForStmt` 保留 | 待执行层迭代原语就绪后处理（P11） |
| `*BreakStmt` / `*ContinueStmt` 保留 | 同上 |
| 最后一条指令是控制转移 | 每个 `BlockStmt.Body` 以 `br` / `call` / `return` 结尾 |

## 二、两类变换

### 2.1 控制流降级

`IfStmt` → 4 个 `BlockStmt`：

```
if(cond) { A } else { B }

→ BlockStmt _if_cond_N  { evalCond; br(cond, iftrue, iffalse) }
  BlockStmt iftrue       { lowerBody(A); call merge }
  BlockStmt iffalse      { lowerBody(B); call merge }
  BlockStmt _merge_N     { }
```

`WhileStmt` → 3 个 `BlockStmt`：

```
while(cond) { body }

→ BlockStmt _while_cond_N  { evalCond; br(cond, body, exit) }
  BlockStmt _while_body_N  { lowerBody(body); call cond }
  BlockStmt _while_exit_N  { }
```

块跳转统一用 `call parent/label`，复用 `HandleCall` 进入——无独立跳转原语。

### 2.2 表达式展开

将嵌套表达式树递归拆平，每层引入一个临时槽，直到每条指令的操作数全为叶节点，kvcpu 可直接执行：

```
result <- a + b * c        （输入：两层嵌套）

→ ./_tmp_1 <- b * c        （临时槽，只赋值一次）
  result   <- a + ./_tmp_1 （操作数全为叶节点）
```

临时槽名格式：`./_tmp_N`（帧局部，`./` 前缀，`N` 单调递增）。
条件表达式同理，临时槽名格式：`./_cond_N`。

## 三、文件职责

单文件 `lower.go`，当前 213 行。

| 函数 | 职责 |
|------|------|
| `File` / `Func` | 入口，遍历所有函数 |
| `lowerBody` | 递归处理 `[]Stmt`，分发各类节点 |
| `lowerIf` / `lowerWhile` | 生成 `BlockStmt` 序列 |
| `evalCond` | 条件求值：简单槽直接用；复合表达式展开并填写临时写槽 |
| `flattenInstExpr` | 表达式树 → 可直接执行的指令序列（递归） |
| `brInst` / `gotoLabel` / `wrapBlock` | 构造辅助 AST 节点 |
| `labelGen` | 单调递增标签生成器，格式 `_prefix_N` |

## 四、禁止项

| 编号 | 禁止 | 理由 |
|------|------|------|
| LW1 | lower import kvspace / keytree / layoutcode / kvcpu | 纯 AST 变换，不感知执行层 |
| LW2 | lower 修改 `FuncSig`（签名） | lower 只变换函数体，签名不变 |
| LW3 | 临时槽名复用或非单调递增 | 每个临时槽只赋值一次；复用导致 kvcpu 读到上一次执行的残留值 |
| LW4 | `lowerBody` 原地修改输入 `[]Stmt` | 纯函数契约；输入 AST 只读，输出是新节点 |
